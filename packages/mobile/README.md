# AIChat Mobile

AIChat Mobile 是 AIChat 的 Android 客户端包。当前处于阶段 5：聊天 MVP。

## 当前范围

- 新增 `packages/mobile` Expo TypeScript App。
- 接入 pnpm workspace。
- 提供 Android 真机可打开的服务器地址配置页。
- 校验 AIChat 服务端根地址格式。
- 通过 `GET /api/settings/health` 检测服务端可达。
- 保存服务端地址，并在 App 重启后恢复。
- 提供修改服务端地址入口。
- 基于已保存的服务端地址登录和注册。
- 使用服务端返回的 Bearer token 调用认证接口。
- 使用 `expo-secure-store` 保存认证 token，并在 App 重启后恢复登录状态。
- token 失效或服务端返回 401 时清理本地登录状态并回到登录页。
- 支持退出登录，并清理本地 token。
- 展示当前用户可访问的会话列表。
- 使用当前服务端的可用模型创建新会话。
- 处理会话列表加载中、空状态和错误状态。
- 点击会话进入聊天页并加载最近历史消息。
- 发送文本消息到 `POST /api/chat/stream`。
- 通过 SSE 实时展示助手流式回复。
- 支持停止生成。
- 支持基础 Markdown 可读渲染。
- 处理聊天历史加载失败、流中断、401、服务端错误和网络不可达。

当前阶段不做图片上传、文件上传、MCP、Skill、知识库、APK 打包、模型选择或管理后台。

## 环境要求

- Node.js 18 或更高版本。
- pnpm 8 或更高版本。
- Android 真机安装 Expo Go，或本机配置 Android Emulator。

## 安装依赖

在仓库根目录执行：

```bash
pnpm install
```

## 启动

在仓库根目录执行：

```bash
pnpm mobile:start
```

然后用 Android 设备上的 Expo Go 扫描终端二维码。

也可以直接尝试打开 Android 设备或模拟器：

```bash
pnpm mobile:android
```

## 验证

1. Android 设备能打开 `AIChat Mobile`。
2. 首次启动显示服务端地址输入页。
3. 输入不完整地址时，页面显示格式错误提示。
4. 输入可达的 AIChat 后端根地址后，点击“检测并保存”能通过健康检查并保存。
5. 关闭并重开 App 后，页面恢复已保存的服务端地址。
6. 点击修改按钮后，可以重新输入并保存服务端地址。
7. 保存服务端地址后，页面显示登录/注册表单。
8. 使用有效账号登录后，进入会话列表页。
9. 关闭并重开 App 后，页面能恢复登录状态。
10. 点击退出登录后，回到登录页。
11. token 失效或服务端返回 401 后，App 清理本地 token 并回到登录页。
12. 登录后能看到当前用户可访问的会话列表。
13. 点击“新建会话”能创建会话，并进入该会话的聊天页。
14. 点击已有会话能进入聊天页并看到最近历史消息。
15. 能发送一条文本消息。
16. 能看到助手回复持续流式更新。
17. 生成过程中点击停止按钮能停止生成。
18. 基础 Markdown 内容可读，包括标题、列表、引用、行内代码和代码块。
19. token 失效或服务端返回 401 后，App 清理本地 token 并回到登录页。
20. 网络或服务端错误不会卡死聊天页，页面会显示错误并保留刷新或继续发送的恢复路径。
